# Indicator Overlay Sweep

This note records the indicator sweep used to decide whether the production
monitor should add an indicator condition on top of the existing probability
grid.

## Decision

Keep one production probability grid and add one candles-only side gate:

```text
momentum = 10_000 * ln(close_t / close_t-1) / vol_30m
```

- `close_t` and `close_t-1` are the last two closed 1-minute candles.
- `vol_30m` is the existing 30-minute realized candle volatility in bps per
  sqrt minute.
- Compute the value separately for Coinbase and Binance, then average available
  sources.
- If `momentum >= 2.0`, allow only Up-leading trades.
- If `momentum <= -2.0`, allow only Down-leading trades.
- If momentum is missing or inside `(-2.0, 2.0)`, leave the existing grid
  decision unchanged.

This is an overlay, not a replacement grid. The current grid still controls
time bucket, volatility bucket, distance bucket, sample-count filtering,
probability, fees, and executable-edge depth.

## Data

The sweep used the local one-year training store generated by the offline
pipeline:

- assets: BTC, ETH, SOL, XRP, DOGE
- sources: Coinbase spot and Binance spot
- label source: dense cross-source 1-minute VWAP table
- window: `2025-05-02T00:11:00Z` through `2026-05-02T00:11:00Z`

The candle database was rebuilt from scratch, gap-filled where needed, and
VWAP was recomputed before the sweep. The production monitor does not use this
database; it keeps only bounded live candles in memory.

## Sweep Shape

The first pass tested distance from SMA, EMA, and rolling VWAP across periods
`20`, `50`, `100`, `200`, and `300`, with volatility-normalized thresholds and
forward horizons of `1`, `2`, `3`, and `5` minutes. Both mean-reversion and
continuation directions were checked.

The second pass tested additional simple indicators that fit the current data
pipeline:

- 1-minute and 2-minute normalized momentum continuation/reversal
- RSI-style normalized pressure bands
- Bollinger-style rolling z-score continuation/reversal
- simple BTC cross-asset confirmation

The selection rule was deliberately conservative: prefer signals that were
global across assets, non-tiny in sample count, non-overlapping in evaluation,
and large enough to plausibly change PnL rather than just move a metric by a
small statistical amount.

## Result

The strongest simple candidate was 1-minute normalized momentum continuation.
Using non-overlapping 5-minute outcomes with threshold `abs(momentum) >= 2.0`:

```text
all assets: n=9,288, hit=57.19%, baseline=50.00%, lift=+7.19pp, z=13.86
BTC:        lift=+7.84pp
DOGE:       lift=+4.88pp
ETH:        lift=+8.48pp
SOL:        lift=+5.86pp
XRP:        lift=+8.99pp
```

Nearby thresholds behaved as expected:

- `1.5`: more trades, weaker lift
- `2.0`: balanced lift and frequency
- `2.5`: stronger lift, materially lower frequency

SMA/EMA/VWAP distance signals were weaker and more regime-sensitive. The best
EMA100 mean-reversion variant was directionally interesting, but the lift was
smaller and the interpretation was less clean. RSI and Bollinger variants were
also plausible but did not beat the normalized momentum rule enough to justify
another production condition now.

## Production Semantics

The implementation uses the momentum signal as a veto against the opposite
side:

- strong positive 1-minute momentum blocks Down entries
- strong negative 1-minute momentum blocks Up entries
- no strong momentum does not change eligibility

This avoids replacing the probability table or adding conditional grids. It is
also easy to audit: every evaluation log and trade record includes the computed
momentum value, source count, threshold, and selected overlay side.

## Caveats

The sweep used historical VWAP-derived candles; live entries use Chainlink RTDS
for the market price line and current price while Coinbase/Binance candles drive
the overlay. That is acceptable for a side gate, but the live logs should be
reviewed after deployment to confirm source alignment and trade-frequency
impact.

Missing candle momentum does not block trades. That is intentional: the overlay
is additive risk control, while the existing runtime vol gate still requires
sufficient candle history for the base grid.
